Hola Git!

Es un sistema de control de versiones.

Comandos.

git reset HEADSacar archivos del area de staging

git rmRemueve los archivos de git de fomra permamente del disco duro si se usa --force, con --cache mueve los archivos a estado de untracked.

git resetReset a una version anterior del proyeto con --hard se borra todo de la base de datos de git con --soft se siguen guardando los archivos en el area de staging.

git checkoutNos permite visitar una version anterior del proyecto sin borrar nada o saltar a otra rama.

git branch TEST Crea una nueva rama de desarrollo

git merge RAMA Junta el contenido de una rama en la rama HEAD de en caso de conflicto se deben resolver.

git remote add origin URL Agrega un repositorio en la nube usando la url.

git fetch Trae los cambios de la base de datos de un repositorio local, para aplicarlos hay que hacer un merge

git push origin master Envia los cambios del repositorio a la url del repositorio en la nube.

git pull origin master --allow-unrelated-histories Trae los cambios del repositorio y los cambia en local.

git remote set-url origin URL Cambiar la url de donde se hace el fetc y el push del repositorio remoto.

git log --all --graph --decorate --oneline Muestra el log con las diferentes ramas de forma visual.

git tag -a V0.1 -m "Comentariio" 1b56489 Agrega un tag al commit declarado en el comando.

git show-ref --tags Muestra los tags y a donde estan asignados. los tags se deben pushear al repositorio git push origin --tags.

git tag -d TAGNAME Borra el tag , para borrarlo en github se usa git push origin :refs/tags/TAGNAME

git show-branch Historia de las ramas show-branch --all
